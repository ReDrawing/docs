Recording messages

It is very useful to record the messages from a stage, and be able to reproduce this messages later. For this, the redrawing package has two stages, Record and Play

Recording messages

The "Record" stage is able to record the messages from various channels.

Configuration

The configuration dictionary have the following propertys:

Key Value Type Use Default Example
file_name String Sets the name of the recorded file. Default create a file with the starting time in the name "record" "my_file"
channels List[String] Input channels that will be recorded [] ["channel_1","channel2"]

Operation

The stage create input channels with the configuration dict, and stores the messages increasing time (time it's arrived at the stage since the epoch) in the file, using pickle library ==Ou outra que funcione melhor==

Playing messages

The "Play" stage is able to play pre-recorded messages using the "Record" stage.

Configuration

The configuration dictionary have the following propertys:

Key Value Type Use Default Example
file_name String Sets the name of the file to play. Not setting raises a warning "" "myfile"
start_time Float Time to start playing the file, useful for skip messages or delay start. Default plays the file from start None 1623100307.6467733
time_factor Float If different from one, speeds up or slow down the playback 1.0 2.0
