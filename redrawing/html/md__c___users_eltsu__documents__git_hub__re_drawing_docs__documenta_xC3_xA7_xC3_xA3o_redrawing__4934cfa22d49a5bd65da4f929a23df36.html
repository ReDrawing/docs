Objetivos: Conseguir melhorar o funcionamento da programação e uso de módulos para a OAK

  • Permitir saídas assíncronas com a câmera (não ocorrem apenas uma vez por ciclo)
  • Deixar mais claro e transparente como operar com a OAK
  • Simplificar o uso de vários modelos em paralelo
