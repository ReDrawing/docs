Python package provinding tools for artistic interactive applications using AI
Introduction

The ReDrawing package, written in Python, allows dynamic creation of pipelines (sequence of operations) focused on useful modules for artistic creations using AI, with the possibility of streaming the data for others ambients, like Processing, via UDP.

It consists of three majors sub-modules:

  • data_interfaces: classes for data storage
  • components: classes for data processing
  • communication: classes for data transmission

And a fourth module, "third_models", with third-party AI modules.



components

The major sub-module, it contains all the data processing classes. Its base class is "Stage". It contains methods for creating input and output channels for communicate with other stages, and for creating configurations for the stage.

When inheriting from Stage, there some points that need to be considered:

  • The constructor method needs to call the super constructor. Without it the data channels will not work.
  • Ideally, the constructor must not perform any heavy computation, only set the channels.
  • For startup operations, it must override the "setup" method. If this is done, the _config_lock attribute must be mannualy defined.
  • Only communicate with other stages using the data channels. With this is possible to create parallel pipelines.

Stages implemented:

Class Description
OAK Manages the OAK camera. With its settings is possible to set a neural network to run on the device
OpenPose_Light Uses a ligthweight version of OpenPose to estimate the persons poses on a image
HandGesture Detects hand gestures on a Bodypose
CameraReceiver Receives a image from a Android device
IMUReceiver Receives the IMU data from a Android device
UKF_IMU Uses a UKF filter to estimate the orientation of a IMU



data_interfaces

The base class from this module is the "Data" class. It contains methods to convert the message to JSON.

The subclasses can contain attibutes with any standard Python type, like int, dict, list; or Numpy ndarrays.

Types of data implemented:

Class Description
Image Stores a image
IMU Stores IMU (accelerometer, gyroscope, magnetometer) sensor
Orientation Stores a 3D quaternion orientation
Bodypose Stores keypoints from a body pose detection model, like OpenPose
Gesture Stores a detected user gesture
ObjectDetection Stores a detected object with its bounding box
