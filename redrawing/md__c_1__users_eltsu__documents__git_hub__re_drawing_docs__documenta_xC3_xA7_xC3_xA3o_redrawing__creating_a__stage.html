Creating a Stage

To create a stage, one should inherit the Stage base class. The super constructor must be called to properly configure the data channels.

process()

All the stage cyclical operations should be in this method. A typicall flow consists of getting the data in the input channels, process and send the output data to the output channels.

Configurations

The stages can have configurations that can be changed in it initialization, or using the change_config method. Each configuration must have a name.

To set the default configurations for a stage, creates a configs_default class dict attribute, with the default values.

On the initialization, its possible to pass a changed configuration dict, with the values that must diverge from the default. Passing it to the super constructor allows to automatically change the object _configs attribute.

If the _config_lock attribute is setted to True, all the configuration changes will be ignored. The default setup() automatically sets it to True.

Data channels

Creation

To create a data channel, preferably in the constructor method, the addInput or addOutput method must be used. It receives the name of the channel and the data type.

Getting and sending messages

To get a message from a data channel, the method _getInput("name_of_channel") must be used. To send, use the method _setOutput(outData, "output")
